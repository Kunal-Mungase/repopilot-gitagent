# Generate README

Your job is to create a clean, useful README for the project.

## Instructions
1. Read available project files.
2. Identify project purpose, features, stack, and setup.
3. Write a README that is simple and useful.
4. If some details are missing, mark them clearly instead of inventing them.

## README sections
- Project title
- Description
- Features
- Tech stack
- Installation
- Usage
- Folder structure
- Troubleshooting
- Future improvements
